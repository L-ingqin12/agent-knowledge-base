# Git Workflow Conventions

Standard branching and commit conventions.

1. Branch naming: `feature/description`, `fix/description`, `chore/description`
2. Commit messages: `type(scope): description` (e.g., `feat(api): add rate limiting`)
3. Open a PR and request review from at least one teammate
4. Consider using the code_review_checklist before requesting review
5. Merge via squash merge to keep history clean
6. Delete branch after merge

Types: feat, fix, chore, docs, refactor, test, perf.
