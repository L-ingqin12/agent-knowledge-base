# SAE 拆解视觉 token 单义性——NeurIPS 2025 论文精读

> Sparse Autoencoders Learn Monosemantic Features in Vision-Language Models
> Mateusz Pach et al. · NeurIPS 2025 · [github.com/ExplainableML/sae-for-vlm](https://github.com/ExplainableML/sae-for-vlm)

---

## 一、问题背景

### 1.1 视觉 token 是黑箱

LLaVA-1.5 的视觉管道：图片 → CLIP ViT-L/14 → 576 个 patch token（每个 1024 维）→ 投影层 → Vicuna-7B。

每个视觉 token 是 1024 维浮点数向量，人类直接看就是一堆数字。模型说「图里有猫」——是因为真的「看到」了猫，还是统计噪声碰巧触发了「猫」这个词？

### 1.2 原始特征是多义纠缠的（Polysemantic）

直接看 CLIP 残差流的单个维度毫无意义——一个维度可能同时编码了「猫」「毛茸茸」「暖色调」三个概念，这叫 polysemanticity。**不是概念本身不能解释，是 1024 维里塞了几百上千个概念，维度太少必然叠加。**

### 1.3 解法：SAE 升维 + 稀疏化

```
输入 x (d维, 纠缠) → 编码器 → z (d×ε 维, 稀疏) → 解码器 → 重建 x̂
```

- 升维（ε = 1~64）：给每个概念独立的位置站
- 稀疏约束：逼模型「用最少的特征解释输入」
- 重建约束：x̂ ≈ x，保证信息不丢失

两个约束缺一不可：只要重建 → z 不稀疏等于没拆；只要稀疏 → z 全零等于没学。

## 二、实验设置

### 2.1 模型与挂载点

| 项目 | 配置 |
|------|------|
| 视觉编码器 | CLIP ViT-L/14-336px |
| SAE 训练位置 | 残差流 CLS token（L11, L17, L22, L23）+ 最终投影层输出 |
| 因果干预使用层 | Layer 22 |
| 训练数据 | ImageNet 训练集（提取各层激活向量，离线训 SAE，无需标注） |

### 2.2 SAE 架构三种变体

| 架构 | 稀疏方式 | 特点 |
|------|----------|------|
| Standard SAE | L1 惩罚 | 基线 |
| **BatchTopK SAE** | 每 batch 只保留 Top-K 激活 | MS 显著更高 |
| **Matryoshka SAE** | 嵌套多级粒度 ℳ = {0.0625ω, 0.1875ω, 0.4375ω, ω} | 一个 SAE 提供多级精度 |

关键结论：**稀疏性 + 宽隐层（大 ε）是提升单义性的两个最强因子。** BatchTopK 比 L1 稀疏更干净，Matryoshka 在一组嵌套结构中同时提供粗/中/细粒度。

### 2.3 扩展因子 ε

ε ∈ {1, 2, 4, 8, 16, 64}。Layer 22 残差流 d=1024 → 最大 65536 维。

## 三、MonoSemanticity Score（MS）—— 核心方法论贡献

### 3.1 直觉

一个单义特征：**激活它的图片应该互相长得像。** 全程无需人工标签——用一个第三方视觉模型当「裁判」。

### 3.2 数学定义

1. **图片相似度矩阵 S**（裁判编码器 E = DINOv2 ViT-B）：
   $$s_{nm} = \frac{E(x_n) \cdot E(x_m)}{\|E(x_n)\| \|E(x_m)\|}$$

2. **神经元 k 的激活向量**：$a^k_n = \phi^k(f_l(x_n))$，取自第 l 层残差流

3. **Min-max 归一化**：$\tilde{a}^k_n = \frac{a^k_n - \min a^k}{\max a^k - \min a^k}$

4. **激活相关性**：$r^k_{nm} = \tilde{a}^k_n \cdot \tilde{a}^k_m$

5. **MS 最终公式**：
   $$MS^k = \frac{\sum_{n<m} r^k_{nm} \cdot s_{nm}}{\sum_{n<m} r^k_{nm}}$$

值域 [-1, 1]。高激活图片权重高，如果它们在 DINOv2 眼里相似 → MS 高 → 神经元单义。

### 3.3 妙处

- 不需要人工语义标签
- 裁判模型（DINOv2）独立于被分析模型（CLIP）
- 用另一个模型的「眼睛」来当客观裁判——避免自评偏差

## 四、关键实验结果

### 4.1 MS 量化对比——两个数量级提升

| 视觉编码器 | 原始 MS | SAE ε=1 | SAE ε=4 |
|------------|---------|---------|---------|
| CLIP ViT-L/14-336px | 0.01 | 0.82 | 0.89 |
| SigLIP SoViT-400m/14-384px | 0.01 | 0.83 | 0.88 |
| WebSSL MAE-300m/14-224px | 0.01 | 0.79 | 0.92 |
| AIMv2 L/14-224px | 0.01 | 0.59 | 0.85 |

**跨架构普遍有效**——四种编码器（对比学习/自回归/MAE/AIMv2）全有效。AIMv2 的 ε=1 只有 0.59，说明预训练方式影响特征的可拆解程度——自监督预训练学出来的表征纠缠模式与对比学习不同。

### 4.2 人类实验——MTurk 验证

| 项目 | 数据 |
|------|------|
| 标注者 | 71 人 |
| 比较次数 | 1000 次（每次 2 组 × 16 张图） |
| 问题 | "哪一组图片更相似、更聚焦于同一个东西？" |
| **总体一致率** | **82.8%** |
| MS 差距极小时（δ < 0.1）| 56.6% |
| MS 差距极大时（δ > 0.8）| **100%** |

关键含义：MS 指标和人类感知高度对齐。MS 说单义的特征，人类看了也觉得单义。不是统计幻觉。

### 4.3 裁判模型鲁棒性

用 DINOv2 ViT-B 和 CLIP ViT-B 分别做裁判，结果一致性很高——MS 不依赖某个特定裁判模型的怪癖。

## 五、因果干预——最关键的铁证

### 5.1 实验设计

1. CLIP layer 22 后挂 Matryoshka BatchTopK SAE（ε=64）
2. 选一个单义神经元 k，强制设定激活值 = 常数 α
3. 解码回去 → 正常过投影层 → 喂给 Vicuna-7B
4. **LLM 一个参数不动**

### 5.2 定量结果（表 2）

| 指标 | 概念插入 | 概念抑制 |
|------|----------|----------|
| 目标成功 | SAE 48.7% vs DiffMean 53.1% | SAE 64.4% vs DiffMean 64.0% |
| **不破坏其他** | SAE **85.8%** vs DiffMean 66.2% | SAE **81.4%** vs DiffMean 38.7% |
| **综合成功** | SAE **42.4%** vs DiffMean 35.8% | SAE **52.5%** vs DiffMean 33.3% |

SAE 在「不破坏其他内容」这个指标上碾压 DiffMean——因为 SAE 特征是稀疏解耦的，改一个只影响一个概念。DiffMean 改一个线性方向经常把无关概念也带偏。

### 5.3 CLIP 相似度验证（表 3）

| 条件 | 图文相似度 |
|------|-----------|
| 下界（随机图文对） | 0.185 |
| 干预前（白图 + 概念文字） | 0.212 |
| 干预后（白图 + 概念文字） | **0.259** |
| 上界（正确图文对） | 0.283 |

SAE 干预把完全无关的白图推到了接近正确图文对的语义空间位置——说明干预确实在语义层面上推了图片表征，而不是搞坏了模型。

### 5.4 定性效果

- 放大「铅笔」神经元 → LLM 写情诗满篇「石墨的心」「削尖的思念」
- 压低「小刀」神经元 → 场景描述中小刀消失，其他物体不变
- 压低「笔记本电脑」神经元 → 同理

### 5.5 这证明了什么

1. **因果性成立**——视觉 SAE 特征不只是和输出相关，在因果上决定了 LLM 的输出
2. **投影层对齐是有效的**——视觉编码器里的语义确实通过投影层传到了 LLM
3. **可以在视觉端控制语言输出**——不需要改 LLM 任何参数

## 六、边界与局限

### 6.1 SAE 暗物质

SAE 重建不是无损的。残差里有结构化信息，不是噪声——是 SAE 还没学会表示的东西。Engels et al. (arXiv:2410.14670) 将这部分正式命名为 "SAE Dark Matter"。**黑箱拆了一角，没有拆完。**

### 6.2 CLS token 的限制

大部分单义性量化分析在 CLS token（全局汇总向量）上做。但 LLaVA 的 LLM 吃的是 576 个 patch token，不是 CLS。CLS 有全局语义但丢失空间分辨率和定位能力——PatchSAE（ICLR 2025, arXiv:2412.05276）在 patch 级表征上补了这个方向。

### 6.3 仅覆盖视觉侧

视觉 token 进入 LLM 后在逐层如何被「翻译」成语言语义——这篇没做。Neo et al. (ICLR 2025, arXiv:2410.07149) 追踪了这个过程：视觉 token 每过 LLM 一层就被消化一点，越往深走越像 LLM 自己的「母语」。

### 6.4 投影层对齐的是 LLM 的语义，不是人类的语义

视觉 token 被对齐到的是 LLM 的内部语义空间，不是人类的概念空间。可能存在 LLM「理解」但人类无法对应到词汇的概念——这些概念 SAE 拆不出来，因为没有人类标签可以验证。

## 七、核心结论

### 7.1 已验证的事实

1. **SAE 在视觉表征上也有效**——四个编码器、两个数量级的 MS 提升
2. **单义性是真实的**——MTurk 82.8% 人类一致率，差距大时 100%
3. **因果性成立**——改一个 SAE 神经元就能控制 LLM 的视觉理解输出
4. **跨架构泛化**——CLIP/SigLIP/MAE/AIMv2 都有效，但预训练方式影响可拆解程度
5. **稀疏性 > 宽度的交互**——BatchTopK（硬稀疏）比 L1（软稀疏）效果更好

### 7.2 方法论贡献

最重要的不是「又训了几个 SAE」，而是建立了完整的验证链：

```
MS 量化指标 → 人类对齐（MTurk）→ 因果干预（改特征改输出）→ 跨模型泛化（4 种编码器）
```

四个环节互相印证，没有单一环节可以单独成立。

### 7.3 架构启示

- **拼接式多模态模型（LLaVA/Qwen3-VL）有可解释性红利**——视觉编码器是独立模块，可以单独分析，不需要碰 LLM
- **原生多模态模型（Gemini）**从第一层就把视觉和语言纠缠在一起——没法单独分析视觉特征
- 「拼接架构最大的隐藏收益不是便宜，是可拆——能拆开看的黑箱，就不算完全的黑箱。」

## 九、一句话记忆标签

> SAE 不是万能翻译器，但它第一次让我们能指着一个神经元说：**这个，代表狗。** ——而且改它，模型就真的看不见狗了。

> 视觉 token 的本质是一张图片的语义骨架——比照片粗糙，比文字丰富，人类看不懂，但 LLM 读得懂。SAE 让我们开始能读懂 LLM 读到的。
